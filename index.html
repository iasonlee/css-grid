I'm working on the perfect grid for a blog site that uses a single page template. The template focuses on the placement of items to make sure that regardless of titles or lengths of different text items, the overall look and feel is almost identical. I'll be making changes and making the CSS smaller and smaller until it is optimized. I'm going to be using a version of the grid myself on my own site.

The grid is set up and ready for things like code snippets: 

hi = ["hi", "look", "at", "this", "code-block"]
for i in hi:
  print(i)
print('hello and welcome to my new site where code can be horizontally scrolled if the lines happen to be very long such as this one.')
print('also vertically scrollable')
bye = ["ok", "goodbye", "now"]
for j in bye:
  print(j)

and even inline TeX writing with Math-Jax. $\forall x\in\mathbb{R},\exists{y}\in\mathbb{R}:x\le y$ completely inline and allows you to continue writing.

Main features

  • Modern theme
  • Scrollable code blocks with highlighting
  • Inline TeX
  • Automatic dark mode / light mode
  • Grid layout that works seamlessly with many types of content and works well on mobile

Callouts can go anywhere on your pages. They can also be easily made into shortcodes on major site processors.

To use the grid in a .md processor, it is simple. You just need to place the form of {{.Content}} inside of the body-text div and set this HTML as your format for your single page layout. Then, cloning the CSS will get everything looking the same. You'll want to update the tag-link (s) with a custom href and name based on your own post folder content. Same with the title, h1 title, box-link and home-link

Clone

If you happen to like the grid you can totally copy it. The easiest way would really be to just set up your individual page layout to follow the layout of this index and then set it to use the styles as well. Then you can keep different themes for other pages if you wish.

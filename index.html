About

I'm working on a perfectly simple grid for a blog site that uses a single page template. The template focuses on the placement of items to make sure that regardless of titles or lengths of different text items, the overall look and feel is almost identical. I'll be making changes and making the CSS smaller and smaller until it is optimized. I'm using a version of the grid for my own site.

Main features

  • Modern theme
  • Easily legible text
  • Scrollable code blocks with highlighting
  • Inline TeX
  • Automatic dark mode / light mode (85%+ coverage)
  • Grid layout that works seamlessly with many types of content and works well on mobile
  • Covers most items used in Markdown for an easy integration.

So, the grid is set up and ready for things like code snippets:

hi = ["hi", "look", "at", "this", "code-block"]
for i in hi:
  print(i)
print('hello and welcome to my new site where code can be horizontally scrolled if the lines happen to be very long such as this one.')
print('also vertically scrollable')
bye = ["ok", "goodbye", "now"]
for j in bye:
  print(j)

and even inline TeX writing with Math-Jax. $\forall x\in\mathbb{R},\exists{y}\in\mathbb{R}:x\le y$ completely inline and allows you to continue writing.

Clone

If you like the grid you can copy it. The easiest way would be to set up your individual page layout to follow the layout of this index and then set it to use the styles as well. Then you can keep different themes for other pages.

To use the grid in a .md processor, it is simple. You just need to place the form of {{.Content}} inside of the body-text div and set this index as your format for your single page layout. Then, cloning the CSS will get everything looking the same. You'll want to update the tag-link (s) with a custom href and name based on your own post folder content. Same with the title, h1 title, box-link.
